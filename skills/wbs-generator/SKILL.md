# WBS 생성 스킬 (1차 기본 구조)

빈 템플릿(`template/WBS_template.xlsx`)에 데이터를 채워 완성된 WBS 엑셀을 만든다.
(사람용 설명·규칙은 README.md에 통합되어 있다.)

## ★ 구조 규칙 (반드시 준수)

- **템플릿은 원본 WBS 샘플을 베이스로 한다.** 스타일·서식·간트(조건부서식 재구성)·
  No 병합헤더는 원본을 따르며, 스크립트는 데이터만 채운다.
- 컬럼: No(A~C, 레벨별 한 칸) · L1(D) · L2(E) · L3(F) · 필요자료(G) · 마감기한(H) ·
  수행사(I) · 담당자(J) · 산출물(K) · 시작일(L) · 종료일(M) · 진행일(N) ·
  총/목표/실 작업량(O~Q) · 목표진척율(R) · 진척율(S) · 상태(T) · 비고(U) · 달력(V~).
- **레벨 L1~L3** (L4 없음). No는 3칸(A=L1, B=L2, C=L3)에 각 자릿수가 레벨 —
  L1=`a00`, L2=`ac0`, L3=`acd` (없는 자리는 0). 예: 100 / 110 / 111 / 112 / 120 / 200.
- **레벨별 행 색**: L1=짙은 회색(흰 글씨), L2=연한 노랑, L3=흰색.
- **L1은 8개로 고정**: `프로젝트 관리` `기획` `디자인` `퍼블리싱` `개발` `테스트`
  `오픈` `안정화`. 모든 WBS는 이 대분류 안에서 작업을 나눈다(해당 단계만 사용).
- **계산은 전부 엑셀 수식**(셀을 고치면 자동 재계산). 규칙:
  - **진행일** = `NETWORKDAYS(시작일, 종료일, 공휴일)` (각 행).
  - **총작업량** = L3는 진행일과 동일, L2 = 하위 L3 합, L1 = 하위 L2 합.
  - **목표작업량** = `총작업량 × IF(기준일<시작,0, IF(기준일≥종료,1,
    NETWORKDAYS(시작,기준일,공휴일)/NETWORKDAYS(시작,종료,공휴일)))`.
  - **실작업량** = 총작업량 × 진척율 (L3), 상위는 하위 합.
  - **목표진척율** = 목표작업량 / 총작업량 (%).
  - **진척율(%)** = **L3는 사용자 입력**(0~1), L1·L2는 실작업량/총작업량 자동.
- ⇒ 사용자가 L3에 넣는 건 **시작일·종료일·진척율·상태**뿐. 작업량은 모두 자동.
- 상위 작업의 시작/종료일 = 자식들의 최소~최대 자동(직접 지정도 가능).
- **상태**는 드롭다운(`대기/진행중/고객검토/수정반영/완료/보류`)이며 상태에 따라
  간트 막대 색이 바뀐다. 수행사·필요자료·마감기한은 비워두는 게 기본.
- **기준일 = 오늘 날짜 자동**(입력 불필요). 간트차트에 세로 기준선으로 표시된다.
  전체 작업 행은 **최대 60개**.
- 시작일·종료일 → 간트차트 자동. 주말·공휴일은 색 표시 + 진행일 계산에서 제외.
- **달력 날짜는 수식 기반**(메타 시작일 M3·종료일 R3 셀 참조). 시트에서 M3/R3를
  고치면 달력·요일·월·간트가 자동 재계산된다. (종료일을 크게 늘리면 오른쪽
  숨겨진 열을 펼쳐야 보임 — 열 숨김은 생성 시점 기준으로 고정)
- **공휴일 시트 자동 생성**: 기간 내 대한민국 공휴일(날짜·명칭). `holidays`
  라이브러리로 음력(설날·추석 등)·대체휴일까지 자동. (제헌절은 비공휴일이라 제외)

## 작동 순서

### 0. 시작 인터뷰 (★ 순서대로 묻고 답을 받은 뒤 생성)
스킬을 시작하면 **아래를 순서대로 묻는다.** (한꺼번에 묻지 말고 대화로, 모르면
제안·기본값 제시) 답을 다 받은 뒤 작업 트리 초안을 만든다.

1. **프로젝트명**
2. **시작일 · 종료일** (전체 기간) — 달력 범위가 된다.
3. **프로젝트 개요** — "무엇을 만드는지" 1~2문장. 작업(L2·L3) 분해의 기준이므로
   가장 중요. (없으면 일반적 작업으로 채운다고 안내)
4. **적용 단계(L1) 선택** — 다음 8개 중 이 프로젝트에 해당하는 것만:
   `프로젝트 관리` `기획` `디자인` `퍼블리싱` `개발` `테스트` `오픈` `안정화`.
5. **작성자 · 버전** — 표지에 들어감. (작성일 = 오늘 자동)
6. **주요 담당자/팀** (선택) — "기획=신지은, 개발=개발사" 식. 담당자 칸을 채운다.
7. **추가 칸 필요 여부** — `필요자료` · `마감기한` · `수행사` 칸을 표시할지 묻는다.
   **기본은 모두 숨김**이고, 사용자가 필요하다고 한 칸만 표시한다.
   (JSON의 `optional_columns`에 표시할 이름만 넣는다. 예: `["마감기한"]`)
8. **상세 수준** — "대분류(L1)만 / L2까지 / L3까지 상세" 중 어느 깊이로 짤지 **매번 묻는다.**

자동 처리(묻지 않음): **기준일 = 오늘**(간트 세로 기준선), **공휴일 = 한국 공휴일 자동**.
사용자가 기획서·요건 문서를 첨부하거나 내용을 주면 먼저 읽어 개요·작업 분해에 반영한다.

### 1. 참고 자료 확인
사용자가 기획서·일정 메모를 **첨부하거나 붙여넣어** 주면 먼저 읽어 초안에 반영한다.
(없으면 개요만으로 분해한다)

### 3. 작업 트리 구성 — 레벨별로 단계적으로 (★ 한 번에 다 묻지 않는다)
작업 전체를 한꺼번에 받지 말고, **레벨을 나눠 "제안 → 확인"을 반복**한다. 각
단계는 한 레벨만 짧게 보여줘 사용자 부담을 줄인다. 매 단계 제가 개요·관행 기반으로
**초안을 먼저 제안**하고, 사용자는 추가/삭제/수정만 한다.

- **(B) L2 제안** — 0단계에서 고른 L1(고정 8종 중 선택분)마다 중분류(L2) 초안을
  표로 제시 → 사용자가 수정·확정. **이때 L3·일정은 묻지 않는다.**
- **(C) L3 제안** — L2 확정 후, 각 L2의 세부작업(L3)을 제시 → 수정·확정.
  (상세수준이 "L2까지"면 이 단계 생략, "대분류만"이면 B·C 모두 생략)
- **(D) 일정·담당자** — 확정된 **말단 작업**에 시작/종료일·담당자(·필요 시 진척율)를
  제안 → 수정. 상위(L1/L2)에는 작업량·일정을 넣지 않는다(자동 합산/자동 계산).

한 단계라도 사용자가 "알아서 해줘"라고 하면 합리적 기본으로 채우되, 생성 직전
전체 트리를 한 번 더 표로 보여주고 최종 확인을 받는다. 임의 확정 금지.

### 4. JSON 작성 후 스크립트 실행
```
cd skills/wbs-generator
PYTHONIOENCODING=utf-8 PYTHONUTF8=1 python scripts/build_wbs.py <입력.json> <출력.xlsx> template/WBS_template.xlsx
```
출력 파일명은 `프로젝트명_WBS_YYYYMMDD.xlsx` 식으로.
- 사전 요구: `pip install openpyxl holidays`
- 템플릿은 원본 WBS 샘플에서 파생한다. 다시 만들려면:
  `python scripts/make_template.py <원본_WBS샘플.xlsx> template/WBS_template.xlsx`

### 5. 결과 안내
생성 경로를 알려주고 "엑셀에서 열면 진척률·간트차트·공휴일이 자동으로 보인다"고 안내.

## 입력 JSON 구조

```json
{
  "project": {
    "name": "프로젝트명",
    "start": "2026-07-01", "end": "2026-10-14",
    "author": "신지은", "author_date": "2026-06-25", "version": "v0.1"
  },
  "tasks": [
    {"level":1,"name":"기획"},
    {"level":2,"name":"요구사항 분석","담당자":"신지은"},
    {"level":3,"name":"요구사항 수집","산출물":"요구사항정의서",
     "시작일":"2026-07-01","종료일":"2026-07-07","진척율":1.0,"상태":"완료"},
    {"level":3,"name":"고객 검토",
     "시작일":"2026-07-08","종료일":"2026-07-10","진척율":0.85,"상태":"고객검토"}
  ]
}
```
- **L3(말단)에는 `시작일·종료일·진척율·상태`만 준다.** 총/목표/실 작업량과
  목표진척율은 엑셀 수식으로 자동 계산되므로 입력하지 않는다.
- L1·L2는 `level·name`(+선택 담당자)만. 작업량·진척율은 자동(하위 합산/비율).
- `holidays`를 생략하면 기간 내 한국 공휴일을 자동 생성한다. 직접 지정하려면
  `"holidays": [["2026-08-15","광복절"], "2026-08-17"]` 형태로 줄 수 있다.
- `optional_columns`(선택): 표시할 추가 칸 이름 배열. 기본은 모두 숨김.
  예: `"optional_columns": ["마감기한"]` → 마감기한 칸만 표시. (필요자료·수행사는 숨김)

### 필드 참고
- 날짜는 모두 `"YYYY-MM-DD"`.
- **상위 작업(L1·L2)**: `level`, `name`, (선택)담당자·상태. 작업량·진척율은 자동.
- **말단 작업(L3)**: `시작일`·`종료일`·`진척율`(0~1)·`상태`. 작업량은 전부 자동 계산.
- `상태`(드롭다운): `대기` `진행중` `고객검토` `수정반영` `완료` `보류`.
  비우면 진척율로 자동 추정. 상태에 따라 간트 막대 색이 달라진다.
- 선택 필드: 담당자·산출물·비고. (원본 컬럼이라 `필요자료`·`마감기한`·`수행사`도
  값을 주면 채워지지만, 기본은 비워둔다.)

## 주의
- 전체 작업 행은 최대 60개.
- 템플릿 파일은 직접 수정하지 말고 항상 새 출력 파일로 저장.
- 진행일·총/목표/실 작업량·목표진척율·(상위)진척율은 모두 수식으로 자동 계산된다.
  사용자 입력은 L3의 진척율과 각 행의 일정·상태뿐이다.
