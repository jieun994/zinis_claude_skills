# qa-master — ISTQB 기반 테스트케이스 생성·QA 스킬

## 뭐 하는 스킬인가요?

국제 소프트웨어 테스팅 자격 표준인 **ISTQB**(CTFL·CTAL·CTEL) 지식을 통째로 담은 QA 전문가 스킬입니다.
기획서나 요구사항 문서를 주면 검증된 테스트 설계 기법(동등분할, 경계값분석, 결정테이블, 상태전이 등)을
적용해 **테스트케이스 엑셀**을 만들어주고, 기획서의 빈틈을 찾는 **QA 검토**와 **테스트 계획 수립**도 해줍니다.

감으로 케이스를 뽑는 게 아니라, 케이스마다 "어떤 기법으로, 어떤 문서 근거로" 만들었는지 기록하기 때문에
빠뜨린 곳과 지어낸 곳이 없는지 바로 확인할 수 있습니다.

## 어떻게 쓰나요?

Claude에게 문서(기획서, 요구사항정의서, IA, 플로우차트 등)를 주면서 이렇게 말하세요.

| 하고 싶은 것 | 이렇게 말하세요 |
|---|---|
| 테스트케이스 만들기 | "이 기획서로 테스트케이스 만들어줘" |
| 테스트 시나리오 뽑기 | "주문 기능 테스트 시나리오 뽑아줘" |
| 기획서 검토 | "이 기획서 QA해줘", "빠진 예외 처리 찾아줘" |
| 테스트 계획 | "이 프로젝트 테스트 계획 세워줘" |

문서가 없어도 말로 기능을 설명하면 만들어주지만, 그 경우 결과물에 "AI 보완"으로 표시됩니다.

## 예시

**입력**: "로그인 기능 테스트케이스 만들어줘" + 기획서 첨부

**결과**: 엑셀 파일 (시트 3개)

| TC ID | 소분류 | 우선순위 | 적용 기법 | 테스트 절차 | 기대결과 | 출처 |
|---|---|---|---|---|---|---|
| TC-LGN-001 | 비밀번호 입력 | 상 | 경계값분석 | 1. 8자리 비밀번호 입력 2. 로그인 | 로그인 성공 | 기획서 p.12 |
| TC-LGN-002 | 비밀번호 입력 | 상 | 경계값분석 | 1. 7자리 비밀번호 입력 2. 로그인 | "8자 이상 입력" 안내 표시 | AI보완(경계값분석) |
| TC-LGN-003 | 로그인 실패 잠금 | 상 | 상태전이 | 1. 잘못된 비밀번호 5회 입력 | 계정 잠금 + 안내 메일 발송 | 기획서 p.13 |

이 엑셀은 만들고 끝나는 문서가 아니라 테스트 기간 내내 쓰는 문서입니다. 실제 테스트를 하면서
**실행결과**(Pass/Fail/Block/미실행 — 셀을 클릭하면 선택 목록이 나옵니다)를 채우고, 실패하면
**실제 결과** 칸에 오류 내용을, **상태** 칸에 처리 진행(신규/수정중/수정완료/재확인완료)을 기록하면
그대로 진척 보고의 원본이 됩니다.

고객이나 개발사에 전달할 문서가 필요하면 "고객 제출용으로 정리해줘"라고 말하세요 —
내부 검증용 표기(적용 기법, AI보완)를 정리한 제출본을 따로 만들어줍니다.

`samples/` 폴더에서 결과 예시 파일 2개를 열어볼 수 있습니다.

- `결과_예시_테스트케이스.xlsx` — 로그인 기능 예시 (TC 12건, 간단한 케이스)
- `결과_예시_화면설계서_숙소예약.xlsx` — 피그마 화면설계서 24페이지를 입력으로 만든 실전 규모 예시 (시나리오 12건, TC 82건)

## 이 스킬이 알고 있는 것

ISTQB 공인 실러버스 3종을 실무 지식으로 정리해 내장하고 있습니다.

- **CTFL v4.0.1** (Foundation): 테스트 7원칙, 테스트 프로세스, 설계 기법 기초, 리뷰 기법
- **CTAL-TA v4.0** (Test Analyst): 블랙박스 기법 10종 심화, 탐색적 테스트, 리스크 기반 우선순위
- **CTAL-TM v3.0** (Test Management): 테스트 계획, 결함 관리, 메트릭, 완료 기준
- **CTEL** (Expert Level): 테스트 전략 3계층, 프로세스 개선 모델(TMMi 등)

※ 본 스킬의 지식 문서는 ISTQB/KSTQB 실러버스를 참고해 실무 관점으로 재정리한 2차 저작물이며,
원문 문서는 [istqb.org](https://istqb.org)와 [kstqb.org](http://kstqb.org)에서 무료로 받을 수 있습니다.
