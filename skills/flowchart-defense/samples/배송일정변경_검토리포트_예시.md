## 1. 보강 플로우차트

```mermaid
flowchart TB
  A([사용자 발화 · 제품명/변경할날짜/변경될날짜]) --> B[주문내역 조회 API]
  B --> B0{주문내역 존재?}
  B0 -- 조회 실패 --> ER["[추가] 일시 오류로 조회가 안 됐어요 안내"]
  ER --> ERq{"[추가] 재시도?"}
  ERq -- 재시도 --> B
  ERq -- 종료 --> ERend([종료])

  B0 -- 없음 --> B1[주문내역이 없어요 안내 + 타쇼핑몰 주문등록 버튼]
  B1 --> Bend([타쇼핑몰 주문등록 전환·종료])

  B0 -- 있음 --> C0{배송 예정·배송 중 상품 존재?}
  C0 -- 없음 --> TS1[배송 중 상품이 없어요 안내 + 주문등록·제품추천 버튼]
  TS1 --> TSend([제품추천·타쇼핑몰 전환·종료])

  C0 -- 있음 --> D[조건에 맞는 상품 리스트 노출]
  D --> D0{"[추가] 발화 조건에 맞는 상품 존재?"}
  D0 -- 0건 --> Dz["[추가] 상품을 못 찾았어요 + 전체 보기 버튼"]
  Dz --> Dall["[추가] 전체 배송 상품 리스트"]
  Dall --> D1
  D0 -- 있음 --> D1([상품 선택])

  D1 --> CL["[추가] 변경 가능 날짜 조회 API"]
  CL --> CLc{"[추가] 변경 가능 날짜 존재?"}
  CLc -- 0건 --> Cz["[추가] 변경 가능한 날짜가 없어요 안내"]
  Cz --> Czend([변경 불가·종료])
  CLc -- 있음 --> Cal[배송일자 캘린더 노출]

  Fu -- 수정 --> R{"[추가] 무엇을 수정?"}
  R -- 상품 --> D
  R -- 날짜 --> Cal
  Fu -- 취소 --> X["[추가] 변경을 취소했어요 안내"]
  X --> Xend([변경 취소·종료])

  G --> G0{변경 결과}
  G0 -- 완료 --> H1[배송일정변경 완료 안내]
  H1 --> J([종료])
  G0 -- 실패 --> F1["[추가/추정] 처리가 지연돼 접수했어요 · 콜백 안내"]
```

## 8. 고객 확인요망

- **변경가능날짜 조회 API(CL)**: 캘린더에 노출할 날짜의 출처 API가 원본에 없음 — 별도 조회 API 필요 여부 **미확정, 고객 확인 요망**.
- **배송 예정·배송 중 판단 기준(C0)**: 상태 코드 정의 — **미확정**.
- **변경 실패 콜백 방식(F1)**: 유선 vs 접수 시스템 — **추정, 확인 요망**.
- **수정 시 날짜 개별 회귀**: 원본이 단일 캘린더라 '날짜' 일괄 회귀로 설계. 대상일/희망일 분리 필요 시 캘린더 2단계 분리 필요 — **미확정**.
