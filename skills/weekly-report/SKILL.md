# 주간보고서 자동 작성 (weekly-report)

> 매주 쓰는 주간업무 보고서를 **기존 PPT 양식 그대로** 채워서 만들어준다. 진척률은 손으로 하던 계산을 그대로 자동화한다.

이 스킬의 사용자는 비개발자다. 전문 용어는 쉽게 풀어서 안내하고, 파일·계산은 전부 대신 처리한다.

## 핵심 개념

- **W1 / W2 두 가지 양식**: W1 = 평소 주간보고(8장). W2 = 월말 마지막 주 보고로, **W1과 똑같이 시작해서 뒤에 "월간 결산" 4장**(전체 진척률 / 주요 성과 / 이슈·리스크 / 익월 목표)이 더 붙는다(12장).
- **베이스 파일 방식**: 지난주 보고서 PPT를 받아 **복사한 뒤 바뀐 부분만 갱신**한다. 그래야 전체 일정(간트)·서식·누적치가 그대로 유지되고, **회사 데이터가 외부로 나가지 않는다**. 지난주 파일이 없으면(첫 주) 내장 빈 템플릿을 쓴다.
- **진척률 입력은 누적 % 4개**: 구분(전체/기획/디자인/퍼블리싱/개발/검증·릴리즈)마다 `지난주 실적`·`이번주 목표`·`이번주 실적`·`차주 목표`를 누적 %로 받는다. 표의 6칸은 스크립트가 계산한다.

## 워크플로우

### Step 1 — W1 / W2 결정
**타입**: prompt

오늘 날짜가 그 달의 **마지막 주**에 속하면 "이번 주는 월말이라 W2(월간 결산 포함)를 추천한다"고 안내하고, 아니면 W1을 기본으로 한다. 사용자가 직접 "W2로", "이번이 마지막 주야"라고 하면 그대로 따른다. AskUserQuestion으로 W1/W2를 확정한다.

### Step 2 — 베이스 파일 확보
**타입**: prompt

지난주 보고서 PPT 경로를 받는다. 사용자가 첨부/경로를 주면 그것을 `base_file`로 쓴다. 첫 주거나 파일이 없으면 내장 템플릿을 베이스로 안내한다:
- W1: `assets/W1_blank.pptx`
- W2: `assets/W2_blank.pptx`

### Step 3 — 지난주 값 읽기 (이어받기)
**타입**: script

베이스가 지난주 실제 보고서면, 먼저 읽어서 이어받을 값을 추출한다:

```bash
python "scripts/weekly_report.py" read "<지난주 보고서.pptx>"
```

출력 JSON에서 활용한다:
- `cumulative_actual` — 구분별 **누적 실적**. 이번 주의 `지난주 실적` 기본값으로 제안한다.
- `task_units` — 업무구분 목록. 입력받을 행을 이 목록에 맞춘다.
- `week_label`(예: "6월 4주차") — 다음 주차 라벨 제안에 참고한다.

### Step 4 — 내용 입력 받기
**타입**: prompt

다음을 대화로 수집한다. 한 번에 붙여넣게 안내하되, 비면 비운 채로 둔다(강제 생성하지 않는다).

1. **진척률(누적 %)** — 구분마다 4개: `지난주 실적` / `이번주 목표` / `이번주 실적` / `차주 목표`. 첫 주라 지난주 실적이 없으면 `0`으로 둔다. 해당 없는 구분(예: 아직 미투입 디자인)은 `-`로 비운다.
2. **주간 실적 / 차주 계획 텍스트** — 업무구분별로 `이번주 실적`과 `차주 계획`(+관련 산출물). 여러 줄은 줄바꿈으로 구분한다.
3. **주요 이슈** — `요청 사항`, `이슈 사항`.
4. **지연 사유 / 캐치업 방안** (있으면).
5. **W2일 때만** — 월간 결산: `월 전체/업무별 진척률`, `주요 성과`, `이슈 및 리스크`, `익월 목표`.

자동 판단이 가능한 것(주차 라벨, 기간, 보고 날짜)은 날짜로 채우되 사용자에게 확인받는다.

### Step 5 — 진척률 자동 계산
**타입**: script (build 단계에 포함)

표 6칸은 누적 % 4개에서 아래 공식으로 계산된다(사용자가 손으로 하던 계산과 동일). 스크립트가 처리하므로 직접 계산하지 않는다:

| 칸 | 계산식 |
|---|---|
| 주간 계획 | 이번주 목표 − 지난주 실적 |
| 주간 실적 | 이번주 실적 − 지난주 실적 |
| 누적 계획 | 이번주 목표 |
| 누적 실적 | 이번주 실적 |
| 차주 주간 | 차주 목표 − 이번주 실적 |
| 차주 누적 | 차주 목표 |

입력에 숫자가 없거나 분모가 안 맞으면 해당 칸은 `-`로 둔다(계산이 보고서 생성 전체를 막지 않는다).

### Step 6 — 보고서 생성
**타입**: script (generate)

수집한 값을 `data.json`으로 정리해 `scripts/` 옆 임시 경로에 저장한 뒤 빌드한다:

```bash
python "scripts/weekly_report.py" build "<data.json>"
```

`data.json` 형식(값이 없는 키는 생략 가능):

```json
{
  "mode": "W1",
  "base_file": "지난주 보고서.pptx 또는 assets/W1_blank.pptx",
  "output_file": "<프로젝트명>_주간보고서_W1_YYYYMMDD.pptx",
  "week_label": "6월 4주차",
  "report_date": "2026.06.30",
  "period": "기간 : 2026. 06. 22 ~ 2026. 06. 26",
  "actual_basis": "06월 26일",
  "week_no": 2, "this_week_range": "6/22~6/26",
  "next_week_no": 3, "next_week_range": "6/29~7/3",
  "progress": {
    "전체": {"prev_actual": 2.1, "this_plan": 7.3, "this_actual": 6.9, "next_plan": 11.8},
    "기획": {"prev_actual": 3.7, "this_plan": 6.5, "this_actual": 7.0, "next_plan": 19.0},
    "디자인": {"prev_actual": "-", "this_plan": "-", "this_actual": "-", "next_plan": "-"}
  },
  "tasks": {
    "프로젝트 관리": {"this_done": "...", "this_output": "...", "next_plan": "...", "next_output": "..."}
  },
  "issues": {"요청 사항": "...", "이슈 사항": "..."},
  "delay": {"지연 사유": "...", "캐치업 방안": "..."},
  "monthly": {
    "month_label": "6월", "next_month": "7월", "period": "2026. 06. 15 ~ 2026. 06. 26",
    "progress": {"전체": [7.3, 6.9, 7.3, 6.9, 23.6, 30.9]},
    "performance": {"기획": {"text": "...", "note": "..."}},
    "issues": [{"issue": "...", "impact": "...", "action": "..."}],
    "goals": {"개발": {"text": "...", "note": "..."}}
  }
}
```

- `progress`/`tasks`의 구분 키는 베이스 표의 구분명과 느슨하게 매칭된다(괄호·공백 차이 허용). `read`의 `task_units`를 그대로 쓰면 안전하다.
- `mode`가 `W2`인데 베이스에 월간 결산이 없으면(예: 지난주 W1 파일) `assets/w2_monthly.pptx`의 4장을 자동으로 #Appendix 앞에 추가한 뒤 `monthly` 값을 채운다.
- `monthly.progress`의 6개 값은 월간표 컬럼 순서(당월계획/당월실적/누적계획/누적실적/익월계획/익월누적) 그대로 들어간다.

빌드 후 출력 경로와 "이번주 누적 진척률 N%"를 안내한다.

발송 메일 초안이 필요하면 jieun-mail 에이전트를 사용한다(주간보고 메일 확정 템플릿 보유). 호출 시 진척률 요약("계획 x% 대비 y%, 차주 누적 목표 z%")과 첨부 파일 목록을 함께 넘긴다.

## 예외 처리 (사용자 경험)

- **항목 일부 누락**: 빈 칸으로 두고 "○○ 칸이 비었어요. 채워서 다시 만들까요?"라고 안내한다. 임의로 지어내지 않는다.
- **진척률 입력이 숫자가 아니거나 비어 있음**: 해당 칸을 `-`로 두고 "진척률은 직접 확인이 필요해요"라고 알린다.
- **첫 주(지난주 파일 없음)**: 빈 템플릿을 베이스로 쓰고 지난주 실적은 `0`으로 둔다. "첫 주라 빈 양식으로 시작했다"고 안내한다.
- **같은 주차 파일이 이미 있음**: 덮어쓰기 전에 확인한다. 기본은 파일명에 주차/날짜가 들어가 있어 충돌이 적다.
- **양식/템플릿을 못 찾음**: 경로를 다시 확인받고, 수집한 입력은 잃지 않게 `data.json`을 보존한다.

## 설정 (가변 요소)

| 설정 | 기본값 | 변경 방법 |
|------|--------|-----------|
| 양식 버전 | 날짜로 W1/W2 추천 | Step 1에서 사용자가 선택 |
| 베이스 파일 | 지난주 보고서 복사 | 없으면 `assets/W{1,2}_blank.pptx` |
| 출력 파일명 | `<프로젝트명>_주간보고서_<W1\|W2>_<YYYYMMDD>.pptx` | data.json의 `output_file` |
| 진척률 표기 | 소수 첫째자리(예: 5.2%) | 스크립트 고정 |

## Scripts
- **`scripts/weekly_report.py`** — `read`(지난주 값 추출) / `build`(보고서 생성·진척률 계산·W2 월간 결산 추가).

## Assets
- **`assets/W1_blank.pptx`** — W1 빈 시작 템플릿(8장).
- **`assets/W2_blank.pptx`** — W2 빈 시작 템플릿(12장).
- **`assets/w2_monthly.pptx`** — W1 베이스로 W2를 만들 때 덧붙이는 월간 결산 4장.

## 핵심 원칙

- **양식을 건드리지 않는다**: 베이스 PPT의 서식·레이아웃·간트는 그대로 두고, 값이 바뀌는 셀만 교체한다.
- **계산은 스크립트에 맡긴다**: 진척률 6칸은 `build`가 계산한다. 직접 암산해 넣지 않는다(불일치 방지).
- **모르면 비운다**: 입력이 없는 칸은 지어내지 말고 비운 채로 두고 사용자에게 알린다.
