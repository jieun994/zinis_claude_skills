# 페이지 유형 & 개발구분 참조

IA의 "페이지 정의 / 요구사항" 칸과 "개발구분(디자인/퍼블리싱/개발)" ● 표시를
일관되게 채우기 위한 참조표다. 서비스에 맞게 응용한다.

## 1. 자주 쓰는 페이지 유형 + 세부 사항 단서

"세부 사항" 칸은 그 유형에서 흔히 떠올려야 할 요소를 단서로 채운다.

| 페이지 유형 | 설명 | 세부 사항에 넣을 단서 |
|------------|------|-----------|
| 디자인 페이지 | 정적 콘텐츠 위주 | 핵심 카피·이미지·다운로드 자료 |
| 메인 페이지 | 서비스 첫 화면 | 배너(롤링 N컷), 바로가기, 최신 콘텐츠 N건, 공지 |
| 리스트 게시판 | 목록형 게시판 | 목록 컬럼, 검색(제목/내용), 정렬, 상단 고정, 페이지네이션 |
| 갤러리 게시판 | 썸네일 그리드 | 그리드 열수, 카테고리 필터, 상세 라이트박스 |
| Q&A 게시판 | 질문·답변 등록형 | 카테고리, 아코디언, 검색, 관리자 답변 |
| 폼메일 프로그램 | 입력 폼 제출 | 입력 필드 목록, 스팸 방지, 관리자 메일·자동응답 |
| 상세 보기 페이지 | 단일 항목 상세 | 표시 항목, 이전/다음, 관련 항목 |
| 리스트 + 상세 | 목록+내용 보기 | 목록 컬럼, 상태 구분, 상세 표시 항목 |
| 회원정보 수정 | 폼 기반 변경 | 재인증, 수정 가능 항목, 비밀번호 변경 |
| 프로그램 페이지 | 처리 로직 중심 | 입력→처리→완료 흐름, 분기·예외 |
| 대시보드 | 지표 위젯 모음 | KPI 위젯, 미처리 항목 바로가기, 기간 필터 |

## 1-1. 정교하게 쓰는 법 (raw → good)

| 칸 | raw (금지) | good (이렇게) |
|---|---|---|
| 페이지 정의 | 게시판 | 리스트 게시판 — 공지 목록·상세, 상단 고정 |
| 페이지 정의 | 로그인 | 로그인 — 계정 인증, 소셜로그인 포함 |
| 세부 사항 | 회원가입 폼 | 아이디(중복검사), 비밀번호(8자+영문/숫자/특수), 이메일(형식검증), 주소(도로명 검색), 휴대폰 본인인증 |
| 세부 사항 | 문의 | 필드(이름·연락처·이메일·유형·내용·첨부), reCAPTCHA, 접수 시 관리자 메일·자동응답 |
| 세부 사항 | 목록 | 번호·제목·작성일·조회수, 제목/내용 검색, 페이지네이션 |

## 2. 개발구분(●) 추천 기준

페이지 유형을 보고 기본값을 자동 제안한다. 사용자가 고치면 그대로 반영.

| 페이지 성격 | 디자인 | 퍼블리싱 | 개발 |
|------------|:---:|:---:|:---:|
| 정적 디자인 페이지(소개·약관) | ● | | |
| 퍼블리싱 필요한 정적 화면 | ● | ● | |
| 게시판·폼·로그인 등 기능 화면 | | ● | ● |
| 데이터 처리·연동·관리자 기능 | | | ● |
| 메인처럼 디자인+기능 모두 | ● | ● | ● |

> 정답이 정해진 건 아니다. 프로젝트 팀 구성(디자이너/퍼블리셔 분리 여부)에 따라
> "퍼블리싱"을 빼고 디자인·개발 2칸만 쓰기도 한다. 시작할 때 한 번 확인한다.

## 3. 권한(역할별 CRUD) — Admin(BIA) 시트용

관리자 화면은 역할마다 할 수 있는 작업이 다르다. 역할(`roles`)을 지정하면 역할마다
**C / R / U / D 4칸**이 생긴다. (예: `에디터 / 운영자 / 최고관리자`)

- **C**=등록(Create) · **R**=조회(Read) · **U**=수정(Update) · **D**=삭제(Delete)
- 입력은 `auth: {"역할명": "CRUD부분집합"}` 으로 준다.
  - `"R"` → 조회만 / `"RU"` → 조회·수정 / `"CRUD"` → 전체 / `""` → 권한 없음
- 허용된 동작 칸에 `O`, 나머지 칸은 연회색으로 표시된다.

> 역할별 권한 추천(일반적인 3단계):
> - 조회 전용 역할 → 대부분 `R`
> - 운영 역할 → 콘텐츠는 `CRUD`, 회원·설정은 `RU` 정도
> - 최고관리자 → 거의 모든 메뉴 `CRUD`

## 4. 특수 영역(공통 요소)

화면 메뉴는 아니지만 IA에 관례적으로 포함하는 항목:

- **Head** — 로고(클릭 시 메인 이동), GNB, 배너 등
- **Footer** — 주소·전화·이메일·SNS 아이콘·Copyright
- **전체 공통사항** — 웹 표준 준수, 반응형, 서브 레이아웃 통일 등 (맨 위 행)

## 5. 2차(Phase 2) 표시

지금 범위 밖이지만 다음 단계에 만들 기능은 행을 회색으로 칠하고 비고에 "2차"라고
적는다. 입력 JSON에서 해당 행에 `"phase2": true` 를 준다.
