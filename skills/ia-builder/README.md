## 1. 무엇을 하나요

- 서비스(웹/앱)의 **메뉴 구조(사이트맵)와 기능정의서를 엑셀로 자동 생성**
- 서비스 설명만 주면 필요한 메뉴를 **1~5단계 트리**로 구성
- 각 화면의 **페이지 유형·개발구분·페이지 수**까지 채워 "사이트 구조 및 기능정의서" 엑셀 생성
- 흔히 IA(Information Architecture, 정보구조)라 부르는 산출물
- 기획 초반, PRD·WBS보다 먼저 만드는 "서비스의 뼈대"

## 2. 어떻게 쓰나요

- Claude에게 **"OO 서비스 IA 만들어줘"** 또는 **"사이트맵 만들어줘"** 라고 요청
- Claude가 아래 순서로 단계적으로 질문 (한 번에 다 묻지 않음)

1. 프로젝트명 · 서비스 개요
2. 일반 사용자만인지, **관리자(Admin) 화면도 있는지** → 있으면 시트 분리
3. **넣을 컬럼 선택** (페이지 정의·개발구분·페이지 수·세부 사항·code·탭)
4. **메뉴 트리 초안** 제시 → 빼고 더하며 확정
5. 확정 시 **엑셀 파일 생성**

- 각 단계마다 초안을 먼저 제시 → 빼고 더하기만 하면 됨
- 참고할 기획서·PRD는 첨부하거나 붙여넣으면 더 정확히 반영

### 이렇게 말하면 됩니다

| 상황 | Claude에게 |
|---|---|
| 새 서비스 기획 | "프리랜서 매칭 플랫폼 IA 만들어줘" |
| 화면 목록이 막막할 때 | "쇼핑몰 앱 만드는데 어떤 화면 필요한지 사이트맵 짜줘" |
| 기획서가 있을 때 | (PRD 붙여넣고) "이거 보고 메뉴 구조 잡아줘" |
| 관리자 화면 포함 | "사용자/관리자 화면 나눠서 정보구조도 그려줘" |

## 3. 만들어지는 결과

엑셀 시트 (표지 → 사용안내 → 개정이력 → 정보구조도)

- **표지** — 프로젝트명·작성자·작성일·버전
- **사용안내** — 파일 보는 법 (컬럼 의미·행 색·권한 표시)
- **개정이력** — 버전·날짜·작성자·변경 내용 (최초 작성 행 자동)
- **정보구조도** — 실제 IA 표 (관리자 화면이 있으면 Front/Admin 시트로 분리)

정보구조도 표 컬럼은 **선택한 것만** 포함:

| 항상 포함 | 선택 포함 |
|---|---|
| Code(첫 열) · Depth1~5 · 비고 | 페이지 정의/요구사항 · 개발구분(디자인/퍼블/개발 ●) · Page 수+총계 · 세부 사항 · Tab/서브 플로우 · 권한(역할별 CRUD) |

- **Depth는 서비스에 맞게 유연** — 얕으면 1~2단계, 복잡하면 4~5단계
- **관리자 화면이 있으면** Front/Admin 시트로 분리, Admin엔 역할별 권한(C/R/U/D) 칸 (C=등록·R=조회·U=수정·D=삭제, 가능한 동작에 O)
- **나중에 할 기능(2차)** 은 회색 행으로 표시
- **Page 수 총계**는 자동 합산(SUM 수식), 정보구조도 시트는 눈금선 꺼짐

결과 미리보기: [samples/결과_예시.xlsx](samples/결과_예시.xlsx)

## 4. 다른 기획 스킬과의 차이

| 스킬 | 만드는 것 | 순서 |
|---|---|---|
| **ia-builder** (이 스킬) | 서비스 **구조**(메뉴·화면) | ① 먼저 |
| show-me-the-prd / feature-spec | 기능 **명세**(PRD) | ② 다음 |
| wbs-generator | 작업 **일정**(WBS) | ③ 마지막 |

- IA로 뼈대 → PRD로 기능 정의 → WBS로 일정 수립 순서
