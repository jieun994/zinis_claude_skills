## 1. 뭐 하는 스킬인가요?

서비스(웹/앱)의 **메뉴 구조(사이트맵)와 기능정의서를 엑셀로 자동 생성**해줍니다.
"이런 서비스 만들 거야"라고 설명하면, 어떤 메뉴가 필요한지 **1~5단계 트리**로 짜주고,
각 화면의 **페이지 유형·개발구분·페이지 수**까지 채워서 **"사이트 구조 및 기능정의서"**
엑셀로 만들어줍니다.

흔히 IA(Information Architecture, 정보구조)라고 부르는 산출물이에요. 보통 기획 초반에
PRD·WBS보다 먼저 만드는 "서비스의 뼈대"입니다.

## 2. 어떻게 쓰나요?

Claude에게 **"OO 서비스 IA 만들어줘"** 또는 **"사이트맵 만들어줘"** 라고 하면 됩니다.
그러면 Claude가 순서대로 물어봐요 (한 번에 다 묻지 않고 단계적으로):

1. 프로젝트명 · 서비스 개요(무엇을 하는 서비스인지)
2. 일반 사용자만인지, **관리자(Admin) 화면도 있는지** → 있으면 시트를 나눠줘요
3. **어떤 컬럼을 넣을지** 골라요 (페이지 정의·개발구분·페이지 수·세부 사항·code·탭)
4. Claude가 **메뉴 트리 초안**을 대화창에 보여줌 → 빼고 더하며 확정
5. 확정하면 **엑셀 파일 생성**

각 단계마다 Claude가 초안을 먼저 보여주니, **빼고 더하는 것만** 하면 됩니다.
참고할 기획서·PRD가 있으면 **Claude에게 첨부하거나 붙여넣어** 주면 더 정확해져요.

### 이렇게 말하면 돼요 (예시)

| 상황 | Claude에게 한마디 |
|---|---|
| 새 서비스 기획 | "프리랜서 매칭 플랫폼 IA 만들어줘" |
| 화면 목록이 막막할 때 | "쇼핑몰 앱 만드는데 어떤 화면 필요한지 사이트맵 짜줘" |
| 기획서가 있을 때 | (PRD 붙여넣고) "이거 보고 메뉴 구조 잡아줘" |
| 관리자 화면 포함 | "사용자/관리자 화면 나눠서 정보구조도 그려줘" |

## 3. 만들어지는 결과 (엑셀 시트: 표지 → 사용안내 → 개정이력 → 정보구조도)

WBS 파일과 같은 스타일로, **표지·사용안내·개정이력** 시트가 먼저 들어가고 그 뒤에
정보구조도(IA) 시트가 붙어요.

- **표지** — 프로젝트명·작성자·작성일·버전
- **사용안내** — 이 파일 보는 법 (컬럼 의미·행 색·권한 표시 설명)
- **개정이력** — 버전·날짜·작성자·변경 내용 (최초 작성 행 자동)
- **정보구조도** — 실제 IA 표 (관리자 화면이 있으면 Front/Admin 시트로 나뉨)

정보구조도 표 컬럼은 **고른 것만** 들어갑니다:

| 항상 들어감 | 골라서 넣음 |
|---|---|
| Code(첫 열) · Depth1~5 · 비고 | 페이지 정의/요구사항 · 개발구분(디자인/퍼블/개발 ●) · Page 수+총계 · 세부 사항 · Tab/서브 플로우 · 권한(역할별 CRUD) |

- **Depth는 서비스에 맞게 유연하게** — 얕으면 1~2단계, 복잡하면 4~5단계.
- **관리자 화면이 있으면** Front / Admin 시트로 나누고, Admin엔 **역할별 권한(C/R/U/D)** 칸.
  (C=등록 · R=조회 · U=수정 · D=삭제, 가능한 동작에 O 표시)
- **나중에 할 기능(2차)** 은 회색 행으로 표시.
- **Page 수 총계**는 자동 합산(SUM 수식), 정보구조도 시트는 **눈금선 꺼짐**.

## 4. 다른 기획 스킬과 어떻게 다른가요?

| 스킬 | 만드는 것 | 순서 |
|---|---|---|
| **ia-builder** (이 스킬) | 서비스 **구조**(메뉴·화면) | ① 먼저 |
| show-me-the-prd / feature-spec | 기능 **명세**(PRD) | ② 다음 |
| wbs-generator | 작업 **일정**(WBS) | ③ 마지막 |

IA로 뼈대를 잡고 → PRD로 기능을 정의하고 → WBS로 일정을 짜는 흐름이에요.
